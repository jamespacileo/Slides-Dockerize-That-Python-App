Docker enables you to run completely isolated applications within one single Linux server.

1 APP = 1 Container

each DB, Web App, Web Server, Message Queue, ... = 1 Container

Host OS = Any Linux OS Distro that supports LXC

Container OS = Virtually any Linux Distro

Wait a sec... that sounds a lot like VMs!

Containers vs VMs!

1. Containers don't need to virtualize hardware

2. Same performance as HOST!

3. Minimal space consumption

Just a flesh wound?

Step 2.

Fire the image up in a container

Example Dockerfile

 
FROM  python:2.7

RUN echo 'deb http://us.archive.ubuntu.com/ubuntu/ precise universe' >> /etc/apt/sources.list
RUN apt-get -y update
RUN apt-get -y install python python-pip build-essential
RUN pip install -e git+https://github.com/mitsuhiko/flask.git#egg=flask
RUN pip install flask-sqlalchemy
RUN apt-get -y purge build-essential
RUN apt-get -y autoremove

ADD . /srv/app/

WORKDIR /srv/app/
RUN flask --app=flaskr initdb

EXPOSE 5000

CMD ["flask",  "--app=flaskr", "run", "--host=0.0.0.0"]
